# Mission Control Edge v0.1.0

`v0.1.0` is the provider-contract foundation release. It makes the
provider-neutral protocol usable without a Mission Control gateway or any
specific harness/runtime dependency.

## Included

- `mission-control.provider.v1alpha1` JSON Schema and OpenRPC documents.
- Public Go `protocol`, `provider`, `conformance`, and `mock` packages.
- `@gocodealone/mission-control-provider-sdk` 0.1.0 for Node 22 and newer.
- External-process conformance runner and Go/TypeScript reference providers.
- Linux and macOS archives for amd64 and arm64 containing `mc-schema`,
  `mc-provider-example`, and `mc-conformance`.
- Source archive, SHA-256 checksums, SPDX 2.3 SBOMs, Apache-2.0 license, notices,
  and GitHub repository-bound artifact attestations.

This release does not contain the edge gateway, durable session runtime,
Ratchet, Herdr, Codex, Claude Code, credentials, or provider-native session
state. It does not make Ratchet or Herdr a platform dependency.

## Candidate gate

The release tag must point to the reviewed merge commit. Before tagging, the
candidate runs the complete Go, TypeScript, generated-schema, security,
cross-language conformance, GoReleaser snapshot, and clean-consumer gates. The
clean-consumer proof is meaningful only after the candidate commit is reachable
through the public repository:

```sh
scripts/clean-consumer-proof.sh --go-ref "$(git rev-parse HEAD)" \
  --npm-spec ./sdk/typescript
```

No pull-request workflow can publish a release or package.

## Consumer verification

GitHub's npm registry requires authentication for downloads. After both tag
jobs succeed:

```sh
export NODE_AUTH_TOKEN="$(gh auth token)"
scripts/clean-consumer-proof.sh \
  --go-ref v0.1.0 \
  --npm-spec @gocodealone/mission-control-provider-sdk@0.1.0 \
  --verify-attestations
```

The proof resolves the public Go tag with no workspace or replacement, packs
the published npm version into a clean directory, verifies provenance and SPDX
attestations from `.github/workflows/release.yml@refs/tags/v0.1.0`, compares the
tarball's SHA-256 digest to both signed subjects, installs it, compiles a
provider, and executes cross-language conformance.

Release digests are observed outputs and are not predicted in this document.
The immutable values remain in the GitHub release, checksums, package version,
SBOMs, and attestations.

## Rollback

Published artifacts, package versions, and the tag are immutable. If v0.1.0 is
defective, revert the code on `main`, publish v0.1.1, and deprecate npm 0.1.0
with an explanation. Never replace release assets or delete and recreate the
tag.
